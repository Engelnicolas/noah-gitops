# NOAH GitOps repository

This directory is a self-contained template for the FluxCD GitOps
repository introduced in NOAH v0.0.9. After running
`noah cluster bootstrap`, FluxCD watches `clusters/production/` and
reconciles everything below it onto the cluster.

```
flux-repo/
├── clusters/production/        # FluxCD entry points (Kustomizations)
│   ├── flux-system/            # Auto-generated by `flux bootstrap`
│   ├── infrastructure.yaml     # → infrastructure/
│   └── apps.yaml               # → apps/  (depends on infrastructure)
├── infrastructure/
│   ├── cilium/                 # CNI + Hubble + Gateway/Ingress
│   ├── cert-manager/           # cert-manager + Let's Encrypt issuer
│   └── external-dns/           # Cloudflare DNS sync
├── apps/
│   ├── authentik/              # SSO provider
│   ├── headlamp/               # Cluster UI (OIDC via Authentik)
│   └── hubble-auth/            # Authentik Proxy Provider for Hubble UI
└── .sops.yaml                  # Encryption rules (Age recipients)
```

## Conventions

* All secrets live in `*.enc.yaml` files and are SOPS-encrypted with
  Age. The kustomize controller decrypts them at apply time using the
  `sops-age` Secret seeded by the bootstrap.
* Two Age recipients are required by `.sops.yaml` so that losing
  `Age/keys.txt` does not lock the operator out — keep the second key
  offline.
* `dependsOn` enforces ordering: `apps` waits for `infrastructure`,
  Authentik waits for `cert-manager` + `external-dns`, etc.

## Forking

This `flux-repo/` directory is intended to be **copied to a separate
Git repository** (the URL passed to `noah cluster bootstrap
--flux-repo`). Keeping it inline in the NOAH source tree is convenient
for review and local testing only; do not point a production cluster
at the NOAH repo itself.
